Scoring Dimensions
The projects that impress judges are the ones that genuinely solve a problem and actually run
Judges evaluate holistically across the following dimensions — no fixed weighting formula.
Bitget reserves the right of final interpretation.
Dimension
What judges look for

FAQ
Q: I don't have a deployment environment — is GitHub alone enough?
Yes, but your README must clearly explain how to run it locally. Judges should be able to get it running without contacting you.
Q: Can I submit backtest screenshots?
No. Backtest results must be accompanied by the code or notebook used to generate them. Judges reserve the right to reproduce the results.
Q: What if my Demo requires a login to access?
You must also submit a demo video (public Twitter link or YouTube) so judges can see the project running.
Q: What if my MuleRun / GetAgent Studio link requires a login?
Same as above — you must also submit a demo video.
Q: I don't have live or paper trading data — can I submit with only a backtest?
Yes, but backtest-only submissions are scored lower than projects with actual run records. Backtests must include the code — screenshots alone are not accepted.
Q: Do Open Innovation projects need to submit a usage record?
It's optional, but preferred. Accepted formats are flexible: Agent output screenshots, sample input/output, API call logs, demo run records, test data files — any of these work.
Q: Can I submit an Open Innovation project that has nothing to do with US stocks?
Yes, but you must clearly describe its connection to crypto trading or AI, and use the UID from your registration.
Q: Is Part 4 of the project description (your take on AI Trading) required?
Optional — it won't affect your base score. But if you have genuine hands-on experience and insights, including them helps judges better understand your project.

5. FAQ
Q: Can I participate without trading experience?
Yes. The
Bitget Agent Hub GitHub repository
provides a complete skeleton — just extend it with your own ideas. Bitget Playbook also lets you run a trading strategy end-to-end without writing code.
No real capital required
— sim trading and backtest records are accepted.
Q: Can I participate solo? How does team registration work?
Absolutely. Solo and team entries are judged equally. Everyone registers as an individual by default — no need to submit teammate information at registration. Final team details will be collected again at the submission stage. Teams can have up to 5 members. Looking for teammates? Post in the official Telegram community.
Q: What if my registration fails?
Contact an admin directly in the
official Telegram community
.
Q: How do I use Bitget Agent Hub?
Three ways, use what fits:
MCP Server
: one command to connect Claude Code / Cursor; then call 58 Bitget trading APIs through natural language
Skill Hub
: 5 plug-and-play perception modules (macro / on-chain / sentiment / technicals / news) — configure and go
Direct tool use
: clone from GitHub for fully custom setups
Detailed setup in Chapter 4.
Q: How do I use Bitget Playbook?
Two manual steps first (create sub-account with your registered UID + contact admin via Telegram to get a Playbook API Key), then paste a single prompt into Claude Code — the Agent handles install → strategy creation → backtest → publish automatically. Detailed steps in Chapter 4.
Q: How do I claim MuleRun Credits? What if I didn't receive them?
Join the official Telegram community, sign up on MuleRun, and enter code
0526BITGET
at
credits.mule.page
to claim 2,000 Credits. One claim per email. If you didn't receive them, DM an admin with your UID.
Q: How do I claim Qwen token credits? What if I didn't receive them?
The first 500 registered teams receive an API Key by email within 24h of registration. It works with Cursor and Codex (
Claude Code not supported
). If you didn't receive it, @ an admin in the official Telegram community with your UID and registration time.
Q: The Qwen API Key doesn't work / can't connect to Qwen. What do I do?
This key routes through a Bitget proxy — it
cannot connect directly to the official Qwen API
. You must configure the correct Base URL (
https://hackathon.bitgetops.com/v1
) as shown in the Qwen Setup Guide in Chapter 4. Alternatively, use your own Qwen account API Key.
Q: How do I qualify for the Community Impact Award?
Repost the Bitget
official interaction post
and publish your own content (project ideas, progress, Showcase, etc.) tagged
#BitgetHackathon
and @ the Bitget AI official account. Submit the post links with your project. Submitting a runnable Demo + qualifying post also earns the Participation Award (+50 USDT per team).
Q: Will connecting Agent Hub with my own API Key trigger auto-trades?
Currently only read access is supported — it syncs positions to a simulated account. Order execution is not fully implemented yet; no real-account trades will be triggered automatically.
Q: What if I hit a technical issue?
Join the
official Telegram community
— the technical team is online throughout for support.
